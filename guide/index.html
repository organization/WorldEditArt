Concept introduction, commands documentation and user tips
NOTE: This guide is written assuming that administrators use the default configuration of WorldEditArt.

WorldEditArt is an innovative plugin. This plugin is not written to "port" the MCPC WorldEdit plugin. This plugin is an entirely original work, except that some commands may follow the tradition of WorldEdit to make it easier for understanding. However, a lot of things are different, so don't rely on your knowledge on WorldEdit.

WorldEditArt introduces a number of new concepts into territory of world editing, as well as introduces a lot of improvements to make world editing easier to understand.

WorldEditArt uses a special pattern about command arguments. It is the best if you understand these, but it doesn't really matter if you don't understand, as long as you can use them.
  1. Switches: If you want to specify that something has to be enabled, you have to specify .property in the command. For example, if you want to indicate that you want a hollow cuboid instead of a filled cuboid to be set, you can type //set .h glass, where h stands for hollow. Properties of different commands will be explained later.
  2. Properties: If you want to specify a specific property and indicate the value, you have to specify ,property value in the command. For example, if you want to indicate that you want to set your anchor as the position of player Steve instead of your position, you can type //anchor ,p , where p stands for player. Again, properties of different commands will be explained later.
  3. If the command asks for an argument but you want to give multiple words, you can put it in quotes "". For example, if you want to copy the selection with the name roller coaster, which contains a space, you have to type //copy "roller coaster" instead of //copy roller coaster - the latter will only be understood as roller instead of roller coaster.

World editing plugins are known as one of the most dangerous plugins - I can't imagine anything more dangerous than world editing plugins, except plugins that let players evaluate code/shell commands directly (BAD BAD BAD IDEA!!!). It can, even if not intended, have devastating destruction on the terrain. A simple example is that if you accidentally create a cuboid that spans over a large area of the map, you would probably delete everything that had been built on that world, and you may just be unable to undo that (although WorldEditArt does try to let you undo, there are still cases that there is irreversible consequence). Therefore, sudo mode is introduced to prevent users from misusing WorldEditArt accidentally, or "my cat clicked on my phone and all blocks to lava" (I'm sure many people have heard that, most of the time excuses).
TL;DR: Sudo mode lets you use WorldEditArt. When you just joined a server, you cannot use any WorldEditArt commands, except //sudo (of course, only if you got that permission e.g. by getting op). When you do //sudo, you unlock the WorldEditArt commands for yourself, which will be locked once you leave the server (or if you specify sudo session length - read the commands section for more information).

We may occasionally encounter these cases:

  • This server has a bad builder. He said he was building a stadium, so I went there and watched. Then he suddenly suffocated me with blocks!
    (The builder tried to set blocks of an area without noticing that a player entered the editing area)
  • I just clicked an extra zero. Why did the selection get so big?

If you have safe mode enabled, WorldEditArt will not change anything for you for blocks that are not in UCZs.
To mark a zone as under-construction:

  1. Make a selection.
  2. Run the command //uc.
To unmark a zone as under-construction:
  1. Walk into a UCZ.
  2. Run the command //uuc.
In traditional world edit plugins, a "selection" refers to a cuboid which you defined by doing //pos1 and //pos2. This usage is supported in WorldEditArt, but it also supports more choices.
Selections in WorldEditArt can theoretically be in any shapes. However, currently, only the most popular three shapes, namely cuboids, spheres and cylinders, can be selected through commands. (Another plugin can use the WorldEditArt API to define new shapes)
A cuboid is a polygon with 6 faces, each parallel to one of the X, Y and Z planes. In simple words, a cuboid can be of any size, but it must stand upright on the ground.
There are two ways to define a cuboid:
The old-fashioned way and most precise way is to use these two traditional commands. Let's see an example. 
  P +--------+ A
   /        /|
S +--------+ |
  |        | |
  |        | + Q
  |        |/
B +--------+ R
In this diagram, we are going to select the cuboid AB. To do so:
  1. Walk to point A
  2. Execute //1 (alias of //pos1)
  3. Walk to point B
  4. Execute //2 (alias of //pos2)
Note that you have to choose two opposite vertexes in all dimensions. For example, AP, AS, AQ and AR won't work. Only AB would select the whole cuboid. Actually, you can think as if you are dragging open a cuboid when you walk from A to B.
On the other hand, you can also use PR or QS, since they are also opposite vertexes.
If you don't want to keep walking around the world, you can tell another player (for example, Steve) to stand at A, then you stand at B, then you can do //1 ,p steve //2 to quickly finish the selection without walking.
Another mechanism is to use bookmarks. If you didn't name your bookmark (your bookmark name is default), you just need to type //1 .b (or //2 .b). If you gave your bookmark a special name, instead of typing .b, type ,b name of bookmark. Note that the period . has been changed to a comma ,.
As it is like in all other commands, if the bookmark name contains space(s), put quotes " around the name. For example: //1 ,b "team resources base".
Another method of creating cuboids is the //grow command. It was already present as //cub grow in an older version of WorldEditArt, but the mechanism is a bit different ‐ the X, Y and Z parameters are now reflected. This command is useful for creating cuboids of fixed sizes. It is simple to use. Just run //grow x y z, and you will be in the center of a selected cuboid of 2x×2y×2z, like this: 
43211234
+------+  3
|      |  2
|  ()  |  1
|  ()  |  1
|      |  2
+------+  3
43211234
This is the front view of the player (noted as () in the diagram) running //grow 4 3 5. Note that the 5 blocks across the Z axis are not visible because this is a front view.
If you had experience in selecting a range of files on a computer, you would understand the concept of this - a cuboid starts at your position, and to the direction you look at, a diagonal shoots.

A ↘--------+
  |\       |
  | \      |
  |  \     |
  |   \    |
  |    \   |

The tiny arrow on the left top corner indicates that a player is looking at that direction, and then a cuboid of this shape formed along the diagonal in \. This cuboid is formed by typing the command //shoot 11 (the length between A and B is 11).
WorldEditArt currently only supports regular spheres. A sphere selection of 5 blocks radius can be made by simply typing the command: //sphere 5 (or //sph 5 for short).
The sphere will take your current position as the center. To make the sphere take another player's position as the center, like the cuboid ,p switch, type //sph 5 ,p steve to use Steve's position as center of sphere, or ,b name of bookmark or .b to use bookmarks. For more details, see the description about cuboids.
WorldEditArt supports setting blocks randomly from a weighted list, but the syntax is a bit different from that in WorldEdit for Bukkit.
A weighted block list means that instead of only replacing blocks with one block type, WorldEditArt will choose random blocks from a list of block types, and you can decide which block type will appear the most, which one the least, etc.
Block types are separated by spaces. Each block type should be in one of these formats: 
1. block
2. block:damage
3. weight/block
4. weight/block:damage
where weight should be replaced by the weight of the block (relative chance to get spawned), and damage should be replaced by the damage value of the block. The default values of them are 1 and 0 respectively. block should be replaced by the block ID or name of the block. A list of block names can be found here.
Examples:
  • If you //set 0.05/glowstone grass on a piece of ground, you are making a grass ground pierced with glowstone blocks (5 out of 105 blocks are glowstone)
  • If you 10/stone 10/wood coal_ore still_water cobweb 0.01/diamond_ore 200/air, it will end up like a skygrid map that isn't in grids but random positions.
